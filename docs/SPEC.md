# EpisodeID — TV Episode Identifier

A web app that identifies TV episode MKV files (e.g. from MakeMKV) by fuzzy-matching their embedded subtitles against reference transcripts, then proposes a rename mapping.

## Stack

- **Backend:** FastAPI (Python 3.12)
- **Frontend:** plain HTML + htmx + vanilla JS/CSS (matches vinyl-recorder's stack — keep it simple, no Node toolchain)
- **Worker:** FastAPI `BackgroundTasks` for v1
- **Deployment:** Docker + docker-compose, image published to ghcr.io
- **Storage:** SQLite for cached reference transcripts; user files stay on disk

## Core flow

1. User points the app at a folder of MKVs (mounted as a Docker volume) and selects a show + season from TVDB search.
2. Backend extracts the first English subtitle track from each MKV using `ffmpeg`. For PGS/VobSub (image-based), runs OCR via `tesseract`.
3. Backend fetches reference transcripts for the selected season from configured providers and caches them in SQLite.
4. For each file, take the first N (~30) dialogue lines after timestamp T (configurable, default 3 min to skip cold open variations) and fuzzy-match against each candidate episode using `rapidfuzz.fuzz.token_set_ratio`.
5. Hungarian assignment (`scipy.optimize.linear_sum_assignment`) to find globally optimal file→episode mapping.
6. Return proposed mapping with per-file confidence + top 3 candidates per file.
7. User reviews in UI, clicks Confirm, backend renames and moves files into the Jellyfin-compatible layout described in "Output format" below.

## Output format

Output must match Jellyfin's expected layout. The renamer creates a series folder with per-season subfolders and moves each file into place. Reference: <https://jellyfin.org/docs/general/server/media/shows/>.

**Directory structure:**

```
<library_root>/
└── <Series Title> (<Year>) [tvdbid-<id>]/
    ├── Season 00/                              # specials
    │   └── <Series Title> - S00E01 - <Title>.mkv
    ├── Season 01/                              # zero-padded to 2 digits
    │   ├── <Series Title> - S01E01 - <Title>.mkv
    │   └── <Series Title> - S01E02 - <Title>.mkv
    └── Season 02/
        └── <Series Title> - S02E01-E02 - <Title>.mkv     # multi-episode
```

**Filename template (default):**

```
{series_title} - S{season:02}E{episode:02} - {episode_title}.{ext}
```

**Rules the renamer must enforce:**

- Season folders use the literal word `Season` followed by a space and a 2-digit zero-padded number. Never `S01`, never `Season 1`.
- Series folder name should not appear inside season folder names (Jellyfin misdetects shows like "Stargate SG-1" when it does).
- Series folder includes year in parentheses when known: `Series Title (1987)`.
- Optional metadata-provider suffix improves matching: `Series Title (1987) [tvdbid-71470]`. Make this a config toggle (`include_provider_id: bool`, default `true`).
- Specials go in `Season 00`. If the metadata provider doesn't have the special listed, name the file descriptively (e.g. `Behind the Scenes.mkv`) rather than forcing an `S00Exx` to avoid wrong-metadata matches.
- Strip or replace these reserved characters in any name component: `< > : " / \ | ? *`. Replace with a space, then collapse runs of whitespace. Also strip trailing periods (Windows-illegal).
- Replace `*` in titles like `M*A*S*H` with the word equivalent (`MASH`) per Jellyfin docs.
- Multi-episode files (one MKV covers two episode entries): use range syntax `S01E01-E02`. Detect by either MakeMKV merging two episodes into one playlist, or by runtime ≈ 2× single-episode mean.
- Two-part episodes that ARE separate files: append ` (1)` and ` (2)` to the title, e.g. `Encounter at Farpoint (1)`. TheTVDB usually has these as separate episode entries already.

**Library root + dry-run:**

- Renamer takes `library_root` from config (env `MEDIA_ROOT`). Files can be in a different folder than where they'll end up — the renamer moves them into `<library_root>/<series_folder>/<season_folder>/`.
- Always offer a dry-run mode that prints the full plan (source → destination) and only writes when explicitly confirmed via `POST /jobs/{id}/apply` with `confirm: true`.
- Use `os.replace` for atomic moves on the same filesystem; fall back to copy+verify+delete across filesystems.
- Never overwrite existing files — if destination exists, abort that file with a clear error and continue with the rest.

**Configurable bits in `.env.example`:**

```
RENAME_TEMPLATE={series_title} - S{season:02}E{episode:02} - {episode_title}.{ext}
SERIES_FOLDER_TEMPLATE={series_title} ({year}) [tvdbid-{tvdb_id}]
INCLUDE_PROVIDER_ID=true
LIBRARY_ROOT=/media/tv
```

## Subtitle providers (pluggable)

`SubtitleProvider` ABC with `fetch_episode_transcripts(show_id, season) -> dict[ep_num, str]`.

- `OpenSubtitlesProvider` — primary, opensubtitles.com REST API
- `ChakoteyaProvider` — Star Trek transcripts from chakoteya.net
- `SubliminalProvider` — wraps the `subliminal` library as a fallback

Cache resolved transcripts in SQLite keyed by `(provider, tvdb_id, season, episode)`.

## API endpoints

- `POST /scan` — body: `{folder, tvdb_id, season}`. Returns job_id.
- `GET /jobs/{job_id}` — status + progress.
- `GET /jobs/{job_id}/results` — proposed mapping.
- `POST /jobs/{job_id}/apply` — body: `{mapping, template}`. Performs renames.
- `GET /shows/search?q=...` — TVDB passthrough.

## Gotchas

- MakeMKV often rips multiple subtitle tracks; pick by language tag, prefer non-forced.
- Some episodes have unusual cold opens. Build in retry with different sample windows if confidence is low.
- Two-parters share dialogue. Use runtime as tiebreaker.
- OpenSubtitles rate limits: cache aggressively.
- Renames always confirmed, never auto-applied. Show a diff first.
