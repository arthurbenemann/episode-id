# Contributing

## Local setup

Requirements: Python 3.12+, `ffmpeg` and `ffprobe` on `PATH` (only needed
for runs against real MKVs — the test suite is fully mocked and runs
without them).

```bash
git clone https://github.com/arthurbenemann/episode-id.git
cd episode-id
python -m venv .venv && source .venv/bin/activate
make install
make test
```

## Make targets

| Target | What it does |
| --- | --- |
| `make install` | editable install with dev extras |
| `make dev` | run the FastAPI app with hot reload on :8080 |
| `make test` | pytest (offline — network and ffmpeg are mocked) |
| `make lint` | ruff check + format check (CI runs exactly this) |
| `make fmt` | ruff format + autofix |
| `make docker` | build the production image |
| `make docker-test` | run the test suite inside Docker |

## Testing notes

- Unit tests for `matcher`, `renamer`, and provider parsers run without
  network or ffmpeg.
- The Chakoteya tests use a captured HTML fixture; the OpenSubtitles tests
  use `httpx.MockTransport` — neither hits a live API.
- API and UI tests share fixtures from `tests/conftest.py`, which mock
  subtitle extraction and transcript fetching so the rest of the pipeline
  runs for real.
- UI screenshots in `docs/screenshots/` are generated by
  `python scripts/screenshots.py` (needs `wkhtmltoimage`); regenerate them
  whenever templates or CSS change.

## Workflow

- Branch from `main`.
- Use [Conventional Commits](https://www.conventionalcommits.org/) for
  commit messages — `feat:`, `fix:`, `chore:`, `docs:`, `test:`,
  `refactor:`, `perf:`. The release pipeline generates `CHANGELOG.md`
  from these.
- Run `make lint && make test` before opening a PR.
- One concern per PR; smaller is better.

## Release flow

Releases are cut from `main`:

```bash
git tag v0.2.0
git push --tags
```

The `release.yml` GitHub Action re-runs the full CI gate on the tagged
commit, then:

1. Generates release notes from conventional commits via `git-cliff`.
2. Builds a multi-arch Docker image (amd64 + arm64).
3. Publishes to `ghcr.io/arthurbenemann/episode-id`.
4. Creates a GitHub Release with the generated notes attached.
